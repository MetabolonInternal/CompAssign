\section{Executive summary}

We evaluate model \textbf{form} (not feature engineering) for RT regression, focusing on:
\begin{itemize}
  \item \textbf{Accuracy}: global RMSE/MAE on realtest.
  \item \textbf{Uncertainty quality}: empirical coverage of nominal 95\% prediction intervals and their average width.
  \item \textbf{Practicality}: training time and ability to support cold-start predictions.
\end{itemize}

We compare two ridge-style model forms (and legacy lasso baselines) for lib208 and lib209 under
\textbf{cap100 training} and \textbf{realtest evaluation}:
\begin{itemize}
  \item \textbf{Ridge (supercategory, sklearn)}: a per-group ridge baseline that writes coefficient summaries and
  an approximate predictive distribution.
  \item \textbf{Ridge (partial pooling, chem-linear)}: a hierarchical ridge model fit per \texttt{species} with priors pooled
  through \texttt{species\_cluster}, and a chemistry-informed compound prior for cold-start.
  \item \textbf{Lasso (supercategory)} baseline from Sally's legacy model bundles (included for context).
\end{itemize}

\textbf{Key result:} \textbf{Ridge (partial pooling, chem-linear)} maintains the strong seen-compound performance of
partial pooling, yields much tighter prediction intervals with coverage close to nominal (0.95), and additionally
enables single-model predictions for unseen compounds via chemistry, but costs \textbf{hours of training time} (ADVI).
\textbf{Ridge (supercategory, sklearn)} matches the same RMSE (ridge-equivalent point predictions) but produces
\textbf{much narrower windows that under-cover} (coverage $\sim$0.92--0.94), which would systematically exclude
true peaks in downstream peak assignment.

\textbf{Recommendation:} Use \textbf{Ridge (partial pooling, chem-linear)} as the primary model form when we can afford
offline training (e.g., periodic retrains), and keep \textbf{Ridge (supercategory, sklearn)} as a fast baseline for
regression checks and quick iteration.

\section{Introduction}

Downstream peak assignment benefits from RT predictions that are both accurate and accompanied by
well-calibrated uncertainty (for candidate windowing). At the same time, we require a model form that:
\begin{itemize}
  \item generalizes to \textbf{new species not present in training} (cold-start), and
  \item remains stable in the \textbf{long tail} of sparse compound/group history.
\end{itemize}

This report summarizes our current cap100 $\rightarrow$ realtest experiments, compares ridge pooling variants to
legacy lasso baselines, and provides a recommendation on model form.

\section{Methods}

\subsection{Data and group definitions}

The production RT CSVs contain rows of curated RT observations with run-level covariates
(\texttt{IS*}/\texttt{RS*}/\texttt{ES\_\*}), and identifiers including:
\begin{itemize}
  \item \texttt{species\_cluster}: supercategory identifier (matrix/supercategory),
  \item \texttt{species}: subgroup identifier nested under \texttt{species\_cluster},
  \item \texttt{comp\_id}: compound id.
\end{itemize}

The stage-1 artifact stores coefficient summaries per group
\[
g = (\texttt{group\_id}, \texttt{comp\_id}),
\]
where \texttt{group\_id} is derived from a configured \texttt{group\_col}. In this work:
\begin{itemize}
  \item \texttt{group\_col=species\_cluster} (supercategory ridge),
  \item \texttt{group\_col=species} (hierarchical ridge; nested within supercategory).
\end{itemize}

\subsection{Feature set}

Let $\bm{x}_i$ be the vector of run-level covariates for test row $i$. For pooling comparisons in this report we
use \textbf{linear anchors only} (no poly2) to avoid confounding ``pooling effect'' with ``feature expansion effect''.

\subsection{Models}

All models are linear-in-features at prediction time and differ in grouping and pooling structure.

\subsubsection{Ridge (supercategory, sklearn): per-group ridge coefficient summaries}

This baseline trains an independent ridge regression for each group
$g=(\texttt{species\_cluster}, \texttt{comp\_id})$ using the same linear run covariates. It writes a production-friendly
artifact of per-group coefficient posterior summaries (\texttt{beta\_hat}, \texttt{beta\_cov}, \texttt{sigma2\_mean})
and uses a Normal approximation to produce per-row prediction intervals. When we match the ridge penalty $\lambda$ to the
Bayesian ridge formulation and use the same evaluation code, the \emph{point predictions} match up to numerical precision, but the
\emph{interval widths and coverage} can differ substantially.

\subsubsection{Ridge (partial pooling): hierarchical priors for \texttt{species} within \texttt{species\_cluster}}

The partial pooling model keeps the subgroup grouping (\texttt{species}) but introduces hierarchical pooling for:
\begin{itemize}
  \item \textbf{intercepts} with compound-aware structure, and
  \item \textbf{slope prior means} (``slope heads'') with supercategory shrinkage.
\end{itemize}

\paragraph{Intercept prior (\texttt{intercept\_prior=comp\_hier\_supercat}).}
For each subgroup $c$ with parent supercategory $s(c)$ and each compound $k$:
\[
  b_{(c,k)} \sim \mathcal{N}\left(t_0 + \mu_c + \alpha_k, \tau_b\right),
\]
with
\[
  \mu_c \sim \mathcal{N}\left(\mu_{s(c)}, \tau_{\mu,s(c)}\right), \quad
  \mu_s \sim \mathcal{N}(0, \tau_{\mu,\mathrm{supercat}}),
  \quad
  \alpha_k \sim \mathcal{N}(0, \tau_{\mathrm{comp}}).
\]

\paragraph{Slope head (\texttt{slope\_head\_mode=cluster\_supercat}).}
We learn a slope mean for each subgroup $\bm{m}_c \in \mathbb{R}^p$ and use it as the prior mean for group slopes
$\bm{w}_{(c,k)}$. We pool slope heads through the supercategory:
\[
  \bm{m}_c \sim \mathcal{N}\left(\bm{m}_{s(c)}, \tau_{w,s(c)}\right), \quad
  \bm{m}_s \sim \mathcal{N}\left(\bm{m}_0, \tau_{w,\mathrm{supercat}}\right).
\]

\paragraph{Inference.}
We use ADVI for scalability. The exported coefficient summaries incorporate learned prior means so pooling shifts
mean predictions as intended.

\subsubsection{Proposed improvement: Ridge (partial pooling) with a chemistry-informed compound prior (chem-linear)}

The base partial pooling model above treats the compound offsets $\alpha_k$ as exchangeable:
\[
  \alpha_k \sim \mathcal{N}(0, \tau_{\mathrm{comp}}).
\]
This is sufficient when compound $k$ is observed in training (we can estimate $\alpha_k$), but it provides no
model-derived way to set $\alpha_k$ for a \textbf{new compound} ($\texttt{comp\_id}$ unseen at training time).

We therefore propose a chemistry-informed prior mean for $\alpha_k$ using ChemBERTa embeddings. This keeps the model
\textbf{single-model} (no postfit regression) while enabling true compound cold-start predictions.

\paragraph{Indexing and identifiers.}
We use:
\begin{itemize}
  \item $i$ for observation rows in the RT CSV,
  \item $c$ for subgroup (\texttt{species}) and $s(c)$ for its parent supercategory (\texttt{species\_cluster}),
  \item $k$ for compound (\texttt{comp\_id}),
  \item $h(k)$ for chemical identifier (\texttt{compound}/\texttt{chem\_id}) associated with compound $k$,
  \item $d \in \{1,\dots,D\}$ for chemistry embedding coordinates ($D=20$ for PCA-20).
\end{itemize}
In code we remap raw ids to dense indices (arrays) via sorted unique id lists; for example, a compound id $k$ becomes an
index $\kappa(k) \in \{0,\dots,M-1\}$ into the $\alpha$ vector.

\paragraph{Chemistry embeddings.}
Let $\bm{e}_{h} \in \mathbb{R}^D$ be the ChemBERTa PCA-20 embedding vector for chemical id $h$.
We center embeddings using the training-compound mean $\bar{\bm{e}}$:
\[
  \bm{z}_{h} = \bm{e}_{h} - \bar{\bm{e}}.
\]

\paragraph{Chem-linear compound prior.}
We introduce a global chemistry-to-compound weight vector $\bm{\theta}_\alpha \in \mathbb{R}^D$ and define:
\[
  \bm{\theta}_\alpha \sim \mathcal{N}(\bm{0}, \sigma_\theta^2 \bm{I}), \quad
  \delta_k \sim \mathcal{N}(0, \tau_{\mathrm{comp}}), \quad
  \alpha_k = \bm{z}_{h(k)}^\top \bm{\theta}_\alpha + \delta_k,
\]
with centering of $\alpha$ and $\delta$ (mean-zero) for identifiability with $t_0$.
Intuitively, $\bm{z}_{h(k)}^\top \bm{\theta}_\alpha$ is a chemistry-driven prior mean shared across compounds, and
$\delta_k$ is a residual compound-specific deviation that is learned for \emph{seen} compounds (so seen-compound accuracy
is maintained), while contributing uncertainty for \emph{unseen} compounds.
The rest of the partial pooling model is unchanged:
\[
  b_{(c,k)} \sim \mathcal{N}\left(t_0 + \mu_c + \alpha_k, \tau_b\right),
  \quad
  y_{(c,k),i} = b_{(c,k)} + \bm{x}_{(c,k),i}^\top \bm{w}_{(c,k)} + \epsilon_{(c,k),i}.
\]

\paragraph{How this enables unseen compounds.}
When $\texttt{comp\_id}$ is unseen at training time, the original model has no learned $\alpha_k$.
The chem-linear form provides a model-derived prior mean:
\[
  \mathbb{E}[\alpha_{\mathrm{new}} \mid \bm{\theta}_\alpha] \approx \bm{z}_{h(\mathrm{new})}^\top \bm{\theta}_\alpha,
\]
so we can score rows for unseen compounds using only their chemical embedding. We propagate additional uncertainty
using the learned $\tau_{\mathrm{comp}}$ for unseen compounds.

\subsubsection{Lasso baselines}

We include one legacy eslasso baseline for context:
\begin{itemize}
  \item \textbf{Lasso (supercategory)}: one lasso per \texttt{species\_cluster}.
\end{itemize}
Lasso ``intervals'' are window-based (\texttt{pred} $\pm$ \texttt{window}) rather than derived from a probabilistic
predictive distribution. We still report their empirical ``coverage95'' for comparison, but it should not be
interpreted as nominal calibration in the same way as ridge models.

\subsection{Evaluation metrics}

Let $y_i$ be observed RT and $\hat{y}_i$ be the point prediction on realtest (units: minutes).
\begin{itemize}
  \item \textbf{RMSE}: $\sqrt{\frac{1}{n}\sum_i (\hat{y}_i - y_i)^2}$.
  \item \textbf{MAE}: $\frac{1}{n}\sum_i |\hat{y}_i - y_i|$.
  \item \textbf{Coverage@95\%} (ridge models): with predicted standard deviation
  $\hat{\sigma}_i = \sqrt{\sigma^2_g + \bm{x}_i^\top \widehat{\mathrm{Cov}}(\bm{\beta}_g)\bm{x}_i}$, we define the
  nominal 95\% interval as $\hat{y}_i \pm 1.96\,\hat{\sigma}_i$ and compute the fraction of test points inside.
  \item \textbf{Mean interval width} (ridge models): $\frac{1}{n}\sum_i 2\cdot 1.96\cdot \hat{\sigma}_i$.
  \item \textbf{Coverage@95\% / width (lasso)}: computed using the legacy window:
  $|\hat{y}_i - y_i| \le w_i$ with width $2w_i$.
\end{itemize}

We stratify by \textbf{training support bin} of each $(\texttt{group\_id}, \texttt{comp\_id})$ group using bins:
\[
\{\le 1,\; 2,\; 3\text{--}5,\; 6\text{--}10,\; 11\text{--}20,\; 21\text{--}50,\; 51\text{--}100,\; >100\}.
\]

\subsection{Interval calibration: sklearn ridge vs partial pooling}

The sklearn baseline and the PyMC partial pooling model are both linear ridge-style models over the same run covariates,
but they differ in how they share information across sparse groups and how they form prediction intervals.

However, the \textbf{prediction intervals} differ because they depend on how we estimate (and propagate) predictive
variance. In particular, the sklearn coefficient-summary artifact estimates a separate noise scale (\texttt{sigma2\_mean})
for each $(\texttt{species\_cluster}, \texttt{comp\_id})$ group, which can lead to \textbf{overly optimistic}
intervals in practice. Empirically this shows up as substantially \textbf{narrower windows} but \textbf{lower-than-nominal}
coverage (0.92--0.94). In a peak-assignment setting where we use the interval as a hard RT gate, this under-coverage
translates directly into a higher \emph{miss rate}: the true peak is excluded from consideration more often.

The same pattern holds across the long tail in Figure~\ref{fig:support_bin}: the sklearn baseline tends to be narrower
but under-calibrated, while PyMC partial pooling maintains near-nominal coverage with modestly wider intervals.

\subsection{Unseen species holdout experiments (production subset)}

The cap100 $\rightarrow$ realtest results above assess generalization across runs, but do not directly answer the
``cold-start'' question for \textbf{new species}: how do models behave when test rows belong to species unseen at
training time?

We therefore run additional \textbf{group holdout} experiments on a deterministic subset of the cap100 production CSV
(clusters 4 and 5, top-5 species per cluster, top-100 \texttt{comp\_id}; seed=42; holdout\_frac=0.2).
We evaluate the following holdout modes:
\begin{itemize}
  \item \texttt{species}: hold out entire species within each \texttt{species\_cluster}.
  \item \texttt{species\_comp}: hold out \texttt{(species, comp\_id)} pairs but keep \texttt{(species\_cluster, comp\_id)}
  seen in training (new compound for a particular species; supercategory has data).
\end{itemize}

\paragraph{Subset construction (deterministic).}
We first select a small production subset to make group holdouts and evaluation fast and repeatable:
\begin{itemize}
  \item Restrict to \texttt{species\_cluster} in \{4, 5\}.
  \item Within each selected \texttt{species\_cluster}, keep the top $N=5$ \texttt{species} by row count.
  \item Globally, keep the top $M=100$ \texttt{comp\_id} by row count.
\end{itemize}
This yields $\sim$95k rows for lib208 and $\sim$98k rows for lib209 in the runs used for this report.

\paragraph{Split definitions (exact setup).}
All splits are created on the subset CSV with \texttt{seed=42} and \texttt{holdout\_frac=0.2} (see
\texttt{scripts/experiments/rt/run\_unseen\_generalization\_prod\_subset.py}):
\begin{itemize}
  \item \texttt{species}: for each \texttt{species\_cluster} $s$, sample
  $\max(1, \lfloor 0.2\cdot|C_s|\rfloor)$ species ids to hold out (leaving at least one species in training per cluster),
  where $C_s$ is the set of species observed in cluster $s$. Test contains all rows with held-out species.
  \item \texttt{species\_comp}: sample eligible \texttt{(species\_cluster, comp\_id)} pairs that have $\ge 2$ distinct
  species observed; for each chosen \texttt{(cluster, comp\_id)} pair, sample exactly one species and hold out that
  \texttt{(species, comp\_id)} pair. This guarantees the corresponding \texttt{(species\_cluster, comp\_id)} group remains
  present in training.
\end{itemize}

We compare:
\begin{itemize}
  \item \textbf{Ridge (partial pooling, chem-linear)} evaluated with an explicit \textbf{unseen-species backoff}: when a
  held-out species makes \texttt{(species, comp\_id)} unseen, we score using a derived \texttt{(species\_cluster, comp\_id)}
  coefficient obtained by aggregating fitted \texttt{(species, comp\_id)} coefficients across training species.
  \item \textbf{Ridge (supercategory, sklearn)} evaluated with a \textbf{comp-id backoff} for unseen
  \texttt{(species\_cluster, comp\_id)} groups.
  \item \textbf{Lasso (supercategory)} from the legacy external bundle (not retrained on these splits; included for context).
\end{itemize}

We do \textbf{not} include the \textbf{Lasso (species)} baseline here because it is not applicable to these splits
and does not score all rows (it depends on a separate local-model mapping and requires a fallback model anyway).

Tables~\ref{tab:unseen_holdout_lib208} and~\ref{tab:unseen_holdout_lib209} summarize these results.
Compound cold-start (unseen chemistry) is handled separately in Section~4.6.

\subsection{Zero-shot chemistry-feature baselines (single model)}

Section~4.5 focuses on \textbf{unseen species} generalization. Here we isolate the \textbf{true chemical cold-start}
scenario where a chemical identifier (\texttt{compound}/\texttt{chem\_id}) is never seen in training. We hold out 20
\texttt{chem\_id} values entirely from training (selected by stratifying held-out chemicals across the RT range;
seed=42) and train \textbf{single models} that can still score those held-out chemicals because chemistry enters as
features (ChemBERTa PCA-20 embeddings), not via per-compound lookup tables.

We compare only:
\begin{itemize}
  \item \textbf{Ridge (partial pooling, chem-linear; PyMC)}: our single-model RT ridge with a chemistry-informed compound
  prior for unseen compounds (and calibrated posterior uncertainty for prediction intervals),
  \item \textbf{MLP (ChemBERTa PCA-20 + cluster interactions)}: a strong non-linear single-model baseline (point prediction only).
\end{itemize}

\paragraph{How to improve unseen compounds without sacrificing seen compounds (single PyMC model).}
The seen-vs-unseen goals are not contradictory, but the chemistry head must be stabilized so it does not
over-extrapolate on held-out compounds. The most direct single-model improvements to try next are:
\begin{itemize}
  \item \textbf{Add a class anchor inside the model}: $\alpha_k = u_{\texttt{class}(k)} + \bm{z}_{h(k)}^\top\bm{\theta}_\alpha + \delta_k$
  with hierarchical priors on $u_{\texttt{class}}$, so unseen compounds can fall back to a class mean when embeddings are weak.
  \item \textbf{Stronger shrinkage for $\bm{\theta}_\alpha$}: reduce \texttt{theta\_alpha\_prior\_sigma} (or use a global shrinkage prior)
  to prevent high-variance zero-shot predictions while keeping $\delta_k$ so seen-compound accuracy is maintained.
  \item \textbf{Revisit the embedding representation}: try more PCA dimensions and/or remove whitening to reduce brittle OOD directions;
  PCA-20 + whitening is a pragmatic compression but may amplify tail components.
  \item \textbf{Allow mild non-linearity}: replace the linear head with a tiny MLP inside PyMC (still single-model) if a linear
  chemistry-to-compound mapping is insufficient.
\end{itemize}

\section{Discussion}

\paragraph{Accuracy vs uncertainty for peak assignment.}
For downstream peak assignment, tighter uncertainty windows (with maintained calibration) directly reduce the set
of candidate peaks per compound, improving precision. Ridge (partial pooling) produces substantially narrower
intervals while keeping coverage close to nominal, and avoids the under-coverage observed with the sklearn baseline.

\paragraph{Why we do not prefer sklearn intervals (as-is).}
The sklearn ridge baseline produces very narrow intervals, but empirically under-covers the nominal 95\% target.
In a peak assignment pipeline that uses RT intervals as a hard filter, under-coverage becomes a direct failure mode:
the true peak is excluded more often. The PyMC partial pooling model achieves a better operating point: narrow windows
with coverage close to nominal, so it reduces candidate load without sacrificing recall.

\paragraph{Cost of hierarchy.}
The benefit of partial pooling comes at a large training cost: $\sim$7.8 hours (lib208) and $\sim$14.3 hours
(lib209) in the original baseline run. With the chem-linear form and cached fixed Cholesky factors, the same ADVI
configuration took $\sim$2.3 hours (lib208) and $\sim$4.0 hours (lib209) on this machine. If we retrain infrequently
(e.g., periodic offline retrains), this may be acceptable;
if we need rapid iteration, Ridge (supercategory, sklearn) remains valuable.

\paragraph{Cold-start behaviour.}
Both ridge forms can provide predictions for new species as long as the row can be assigned to an existing
\texttt{species\_cluster}. The hierarchical model additionally defines a principled prior over unseen \texttt{species}
within a supercategory, and Section~4.5 shows it can score unseen-species rows reliably via an explicit backoff derived
from fitted training coefficients. For truly unseen chemicals, Section~4.6 evaluates single-model cold-start via the
chem-linear compound prior; the next improvements focus on stabilizing the chemistry head (class anchoring, stronger
shrinkage, and possibly mild non-linearity) while keeping the residual $\delta_k$ so seen-compound performance is
maintained. In practice, cold-start deployments may still benefit from collecting a small number of anchor labels per
new matrix to avoid extreme extrapolation.

\paragraph{Lasso baselines.}
Lasso (supercategory) is generally weaker in RMSE than ridge candidates. Its interval is a fixed window, so coverage
should be interpreted as ``fraction inside the window'' rather than probabilistic calibration.

\paragraph{Why we exclude the Lasso (species) baseline.}
The legacy ``local models'' lasso scores only a subset of rows and therefore requires a fallback model anyway.
On realtest it scores 51.2\% of lib208 rows (RMSE 0.011425) and 82.7\% of lib209 rows (RMSE 0.007741), so we do not
include it as a primary candidate in the selection tables.

\section{Conclusion and recommendation}

Based on current cap100 $\rightarrow$ realtest results, \textbf{Ridge (partial pooling, chem-linear)} is the best overall
model form for peak assignment: it improves RMSE and substantially tightens predictive windows with near-nominal
coverage. The main downside is training time (hours).

We recommend:
\begin{itemize}
  \item \textbf{Primary}: Ridge (partial pooling, chem-linear) as the default going forward (seen performance maintained,
  and enables single-model compound cold-start via chemistry).
  \item \textbf{Fallback / fast iteration}: Ridge (supercategory, sklearn) as a fast, stable baseline and safety net.
  Avoid using its intervals for gating unless we explicitly calibrate them to hit nominal coverage.
\end{itemize}

\section{Future extensions}

\begin{itemize}
  \item \textbf{Peak-assignment integration:} use the predictive distribution as a likelihood term (not only a hard gate)
  and tune operating points explicitly in terms of candidate count vs miss rate.
  \item \textbf{Calibration as a first-class step:} fit a small global (or per-supercategory) predictive-scale calibration
  on a held-out validation set to hit nominal coverage while minimizing window width.
  \item \textbf{Active learning loop closure:} choose which species/compounds to curate next based on assignment ambiguity
  (high candidate count), long-tail support, and calibration failures; retrain periodically and measure end-to-end impact.
  \item \textbf{Instrument transfer:} add an explicit transfer layer (e.g., per-instrument offset/scale, or hierarchical
  run-drift components) so new instruments can adapt using limited anchor data without losing calibration.
  \item \textbf{Cold-start matrices and unseen chemicals:} use chemistry-informed priors for compound effects (e.g.,
  the chem-linear single-model prior in this report) and test richer embedding maps (more PCA dimensions, non-linear heads)
  to further improve true zero-shot accuracy while maintaining calibration.
  \item \textbf{Training speed and operational simplicity:} reduce ADVI wall time (structured VI, fewer parameters, better
  initialization) and provide clean production scripts with sensible hardcoded defaults for reproducibility.
\end{itemize}
